# Local Authentications: CLI vs Application Default Credentials (ADC)

One of the most common points of confusion in local Google Cloud development is that your machine actually maintains **two entirely separate authentication "buckets"**. They serve different purposes and can easily fall out of sync.

## Bucket 1: The CLI Tool Account

- **Command:** `gcloud auth login`
- **What it is:** The account you use to type commands into your terminal (like `gcloud projects list`, `gcloud sql...`, or `gcloud iam...`).
- **Purpose:** Acting as "You" (the human developer) to administer the project infrastructure directly from the command line.
- **How to check it:** Run `gcloud config get account`.

## Bucket 2: Application Default Credentials (ADC)

- **Command:** `gcloud auth application-default login`
- **What it is:** A hidden JSON credential file generated by the CLI, but **meant exclusively for your code/SDKs** (like Node.js, Python, `@google-cloud/firestore`, etc.).
- **Where it lives:** Usually at `~/.config/gcloud/application_default_credentials.json` (or `%APPDATA%\gcloud\application_default_credentials.json` on Windows).
- **Purpose:** Providing a seamless fallback identity for your Google Cloud SDKs when `GOOGLE_CLOUD_CREDENTIALS` is not explicitly set in your `.env`.

## Why They Fall Out of Sync

When you run `gcloud auth login` to change your terminal user, **it does NOT automatically update your ADC bucket**.

If you previously ran `application-default login` using a different email address (like a work email or an old test account), your Next.js server code will continue logging into Firestore as that _old_ account—even if your terminal says you are logged in as the correct owner user.

**The Golden Rule:** If you can run a `gcloud` command successfully in your terminal, but your server code (e.g., Next.js app) throws a `PERMISSION_DENIED` error locally, your two buckets are likely out of sync.

**The Fix:** Force your Application Default Credentials to update by running:

```bash
gcloud auth application-default login
```
